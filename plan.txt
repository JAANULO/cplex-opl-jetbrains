Zadanie 1: Własna ikona plików .mod i .dat
Cel: Zmiana nudnej, domyślnej ikony tekstowej na dedykowaną ikonę OPL.
Pojęcie: SVG to wektorowy format pliku graficznego. IDE go używa, aby ikona nie była rozmazana na monitorach wysokiej rozdzielczości.

Dodaj plik graficzny:

W swoim projekcie wejdź do folderu src/main/resources/.

Kliknij prawym przyciskiem myszy na resources -> New -> Directory. Nazwij go icons.

Skopiuj do tego folderu dowolną małą grafikę (najlepiej plik .svg 16x16 pikseli). Zmień jej nazwę na opl_file.svg.

Stwórz klasę ładującą ikonę:

Przejdź do src/main/kotlin/com/github/cplexopl/.

Utwórz nowy plik Kotlin: OplIcons.kt.

Wklej kod:

Kotlin
package com.github.cplexopl
import com.intellij.openapi.util.IconLoader

object OplIcons {
    @JvmField
    val FILE = IconLoader.getIcon("/icons/opl_file.svg", OplIcons::class.java)

Zadanie 2: Dodanie obsługi skrótu Ctrl+/ (Komentowanie)
Cel: Po zaznaczeniu kodu i wciśnięciu Ctrl+/, IDE automatycznie doda // lub /* */.
Pojęcie: Commenter to wbudowany interfejs IntelliJ. Programujemy go, podając mu tylko "wzory" znaków, których dany język używa do komentarzy.

Utwórz folder na nowe funkcje:

Przejdź do src/main/kotlin/com/github/cplexopl/.

Utwórz nowy pakiet (folder): prawy przycisk myszy -> New -> Package -> wpisz features.

Utwórz klasę Commentera:

Wewnątrz folderu features utwórz plik OplCommenter.kt.

Wklej kod:

Kotlin
package com.github.cplexopl.features
import com.intellij.lang.Commenter

class OplCommenter : Commenter {
    override fun getLineCommentPrefix() = "//"
    override fun getBlockCommentPrefix() = "/*"
    override fun getBlockCommentSuffix() = "*/"
    override fun getCommentedBlockCommentPrefix() = null
    override fun getCommentedBlockCommentSuffix() = null
}
Zarejestruj funkcję w IDE:

Otwórz plik src/main/resources/META-INF/plugin.xml.

W sekcji <extensions defaultExtensionNs="com.intellij"> dopisz tę linijkę:

XML
<lang.commenter language="OPL" implementationClass="com.github.cplexopl.features.OplCommenter"/>
Zadanie 3: Podświetlanie par nawiasów
Cel: Gdy postawisz kursor obok {, (, lub [, IDE podświetli zamykający odpowiednik }, ), ].
Pojęcie: PairedBraceMatcher odczytuje Twoje tokeny (zdefiniowane w lexerze) i wiąże je w pary wewnątrz drzewa struktury kodu (PSI).

Utwórz klasę Matchera:

W folderze features utwórz plik OplPairedBraceMatcher.kt.

Wklej kod (korzysta on z Twojego wygenerowanego OplTypes):

Kotlin
package com.github.cplexopl.features

import com.github.cplexopl.psi.OplTypes
import com.intellij.lang.BracePair
import com.intellij.lang.PairedBraceMatcher
import com.intellij.psi.PsiFile
import com.intellij.psi.tree.IElementType

class OplPairedBraceMatcher : PairedBraceMatcher {
    private val pairs = arrayOf(
        BracePair(OplTypes.LBRACE, OplTypes.RBRACE, true), // Klamry {}
        BracePair(OplTypes.LPAREN, OplTypes.RPAREN, false), // Nawiasy ()
        BracePair(OplTypes.LBRACKET, OplTypes.RBRACKET, false) // Nawiasy kwadratowe []
    )

    override fun getPairs(): Array<BracePair> = pairs
    override fun isPairedBracesAllowedBeforeType(lbraceType: IElementType, contextType: IElementType?) = true
    override fun getCodeConstructStart(file: PsiFile, openingBraceOffset: Int): Int = openingBraceOffset
}
Zarejestruj funkcję w IDE:

Otwórz plugin.xml.

W sekcji <extensions> dopisz tę linijkę:

XML
<lang.braceMatcher language="OPL" implementationClass="com.github.cplexopl.features.OplPairedBraceMatcher"/>
